# SecOps Agent

> 基于 AgentScope v2 框架的安全运营智能体项目。
> AgentScope 文档：https://github.com/agentscope-ai/agentscope/blob/main/README_zh.md

---

## 项目是什么

一个运行在 Docker 里的 **安全运营智能体**。它能做：

1. **接收指令、事件** — 用户对话输入、接入外部安全设备（防火墙、IDS、EDR 等）的告警和日志
2. **自主规划** — 收到输入后，通过自主推理决定查询操作和后续处置计划以及期望的目标，选定预设skill，判断工具调用需求。
3. **处置建议与执行** — 给出结论和建议，或选定参数调用安全工具，执行权限可人工修改，（模仿plan、default、allow的权限划分方法）

过程的每一步展示在前端界面上——模型在思考什么、调用了哪个工具、查到了什么以及需要调用什么工具。

## 核心技术栈

### AgentScope V2 — 统一的 Agent 框架

阿里通义实验室开源的多智能体框架，提供从 Agent 推理到服务部署的完整链路：

- **ReAct 推理引擎**：Reasoning → Act → Observe 的循环——由用户/平台事件自动输入，先思考需要什么信息，再调用工具获取，根据结果继续思考，多轮迭代直到得到结论/达到事件处置的目标。
- **工具系统**：Agent 调用的工具大部分是外部程序，运行在 Workspace 内。AgentScope 提供适配器机制将沙箱能力自动包装为 Tool，不需要为每个外部工具手写 Python 包装函数
- **模型接口**：对接大模型（Qwen）
- **记忆系统**：短期记忆存对话上下文，结构化压缩保留任务目标和关键发现，避免长对话丢失上下文
- **权限系统**：原生支持工具调用、文件读写、命令执行的三级控制（允许 / 拒绝 / 人工审批），是本项目风险分级审批门的基础设施
- **事件系统**：流式输出模型调用、工具执行、用户确认等事件
- **Middleware 机制**：在模型调用、工具执行等关键节点插入自定义逻辑（如安全策略检查），无需修改框架源码

### Agent 组织方式

- **主 Agent**：负责复杂任务的分解和路由。通过 AgentScope 的 Agent-as-Tool 机制，子 Agent 可以像工具一样被主 Agent 调用
- **子 Agent**：各自管理一个特化的固定工作流，被主 Agent 按需调用。用户也可以在不同 session 中直接对话某个子 Agent，跳过主 Agent 调度
- **工具接入**：AgentScope 原生支持 MCP 协议。外部工具作为 MCP Server 运行，Agent 通过 Toolkit 发现并调用，工具可独立部署、用任意语言实现

### 沙箱如何连接外部设备和网络

安全运营场景下，工具需要触达外部安全设备（防火墙、交换机、EDR 端点）。AgentScope 的 Workspace 抽象下，Docker Workspace 中的容器天然可通过 NAT 访问外部网络。需要额外处理的是：目标设备在隔离内网时的网络路由配置、设备凭证的安全注入方式、生产环境的网络出口白名单。这些属于运维配置层，不是框架限制。

### Workspace 详解

Workspace 是 AgentScope 的执行环境抽象层，核心职责是**让 Agent 的代码与它实际运行的位置解耦**。同一套 Agent 逻辑，切换一个配置参数就可以在本地进程、Docker 容器、云端沙箱中运行。

在本项目架构中，Workspace 位于 MCP 工具层之下——Agent 决定调用什么工具，MCP Server 提供具体实现，Workspace 负责给工具执行提供隔离的、可管理的运行环境。

**生命周期管理**：Workspace 按用户会话绑定。同一 session 内的多次请求复用同一个 Workspace 容器，保持文件状态和进程上下文。会话过期或空闲超时后自动销毁回收。AgentScope 还支持容器池预热——提前创建好空闲容器，请求到来时直接分配，减少冷启动延迟。

**三种模式对比**：

| 模式 | 运行位置 | 适用阶段 |
|------|---------|---------|
| 本地 Workspace | 宿主机进程内直接执行 | 开发调试、快速迭代 |
| Docker Workspace | 独立容器，挂载 docker.sock 管理 | 正式运行、需要隔离 |
| 云 Workspace |  |  |

切换方式：修改配置中的执行后端参数，Agent 代码不需要改动。

### 技术栈选择

| 考量 | 决策 | 优势 |
|------|------|------|
| 大模型 | 通义千问 (Qwen) | 与 AgentScope 同属阿里生态 |

| 权限控制 | AgentScope 原生权限系统 | 工具/命令/文件三级审批 |
| 前后端通信 | AgentScope 事件系统 + SSE | 原生输出 |
| 会话 | Redis |  |
| 前端 | React或Vue + TypeScript + Vite SSE 直连后端 | |

## 远期方向

- **事件驱动**：安全设备告警自动唤醒 Agent，而非等用户手动输入
- **双模工作**：已知事件走人工预设 Playbook（确定性），未知事件走 Agent 自主 ReAct 推理（灵活性），两者互补而非替代
- **工具抽象层拓展**：Agent 面向能力接口编程，根据新添加的工具或是需要添加的能力解耦构建。
